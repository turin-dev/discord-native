# Wiki Repair

## Overview

- 이 스킬은 부족한 위키를 채우는 스킬이 아니다.
- 참조점으로 믿기 어려운 위키를 다시 신뢰 가능한 상태로 복구하는 스킬이다.
- 시작 전에 `루트의 AGENTS.md`를 읽고 이 저장소의 링크, frontmatter, source-of-truth 규칙을 다시 맞춘다.
- 목표는 위키를 더 많이 쓰는 것이 아니라, 위키를 다시 믿고 읽을 수 있게 만드는 것이다.

## When to Repair

- 코드와 위키 설명이 충돌한다.
- 위키 문서끼리 같은 대상을 다르게 설명한다.
- 허브, 링크, 인덱스 구조가 깨져 탐색 경로가 망가져 있다.
- 오래된 설명이 현재 구조를 오해하게 만든다.
- 이미 있는 문서가 잘못된 방향으로 길잡이 역할을 하고 있다.

## Repair Moves

- 설명 정정
  - 코드, 설정, 최신 문서를 근거로 틀린 설명을 바로잡는다.
- 링크·허브 복구
  - `_Index`, `Home`, 상호 링크, 고아 노트 연결을 복구한다.
- 충돌 해소
  - 같은 내용을 다르게 말하는 문서를 정리해 하나의 설명 체계로 맞춘다.
- 불확실성 재배치
  - 확정할 수 없는 내용은 단정하지 않고 `Open Questions`나 미확인 섹션으로 되돌린다.

## Boundary Rules

- `wiki-query`는 읽기와 탐색이다. `wiki-repair`는 복구다.
- `wiki-sync`는 현재 작업 반영이다. `wiki-repair`는 참조점 복구다.
- `wiki-expand`는 새 구조를 여는 일이다. `wiki-repair`는 기존 구조를 바로잡는 일이다.
- 단지 내용이 부족하다는 이유만으로 `wiki-repair`를 쓰지 않는다.
- 새 자리가 꼭 필요해지는 순간부터는 `wiki-expand` 경계다.

## Exit / Verification

- 틀리거나 충돌하던 설명이 하나의 근거 체계로 정리됐다.
- 허브와 링크가 다시 탐색 가능한 상태다.
- 확정할 수 없는 내용은 불확실한 상태로 올바르게 되돌려졌다.
- 필요하면 후속 `wiki-lint`로 넓은 구조 점검을 이어갈 수 있다.
