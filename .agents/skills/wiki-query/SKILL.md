# Wiki Query

## Overview

- 이 스킬의 목적은 코드 탐색을 위키 기반 탐색으로 바꾸는 것이다.
- 파일 단위로 무작정 뒤지기 전에, 위키 허브와 핵심 문서를 따라가며 구조를 잡는다.
- 위키를 실제로 수정할 가능성이 보이면 `루트의 AGENTS.md`도 같이 읽어 위키 운영 규칙을 맞춘다.
- 위키가 충분하면 코드로 내려가지 않는다.
- 위키가 부족하면 그 지점만 코드와 설정 파일로 확인한다.
- 위키 품질 문제가 보여도 여기서 직접 복구하지 않는다. 복구 분기는 메타 스킬이 맡는다.

## Entry Search Order

1. 사용자 요청에서 주제를 먼저 잡는다.
   - 앱/패키지
   - 흐름
   - 계약
   - 결정
   - 런북
   - 조사/미확인 사항
2. 주제가 모호하면 `루트의 Home.md`에서 시작한다.
3. 주제가 분명하면 가장 가까운 허브로 바로 들어간다.
   - 아키텍처: `루트의 02 Architecture/`
   - 앱/패키지: `루트의 03 Modules/`
   - 실행 흐름: `루트의 04 Flows/`
   - 계약/모델/환경: `루트의 05 Contracts/`
   - 구조적 판단 근거: `루트의 06 Decisions/`
   - 운영/디버깅: `루트의 07 Runbooks/`
   - 조사/보류 이슈: `루트의 08 Investigations/`, `루트의 Open Questions.md`
4. 허브에서 관련 개별 노트로 내려간다.
5. 용어가 낯설거나 이름이 모호하면 `루트의 Glossary.md`를 본다.
6. wiki 외 기존 `docs/` 문서는 위키가 부족할 때 보조 자료로만 쓴다.
7. 위키를 곧바로 갱신해야 하는 작업이라면 `루트의 AGENTS.md`를 운영 규칙으로 같이 읽는다.

## Reading Strategy

- 허브부터 읽고 개별 노트로 내려간다.
- 파일 목록보다 책임, 경계, 흐름, 런타임, 계약을 먼저 읽는다.
- 같은 스레드에서 이미 읽은 노트는 불필요하게 다시 펼치지 않는다.
- 지금 작업과 직접 연결된 노트만 따라간다.
- 위키만으로 충분한 맥락이 잡히면 코드로 내려가지 않는다.
- 위키가 부족한 부분만 코드와 설정 파일로 확인한다.
- stale 설명이나 충돌 조짐을 발견해도 이 스킬 안에서 고치지 않는다. 메타 스킬이 `wiki-repair` 여부를 정하도록 신호만 남긴다.
