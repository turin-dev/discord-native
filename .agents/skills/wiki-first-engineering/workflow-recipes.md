# Workflow Recipes

## 기능 추가

1. `wiki-query`로 관련 허브와 핵심 노트를 읽는다.
2. 구현에 들어간다.
3. 변경된 책임/흐름/계약을 `wiki-sync`로 반영한다.
4. 여러 문서를 만졌으면 `wiki-lint`로 마감한다.

## 버그 수정

1. `wiki-query`로 현재 설명과 관련 런북/흐름을 읽는다.
2. 버그를 수정한다.
3. 기존 설명이 바뀌었거나 디버깅 포인트가 달라졌으면 `wiki-sync`로 갱신한다.
4. 위키 설명 자체가 틀렸다면 `wiki-repair`를 쓴다.

## 구조 변경

1. `wiki-query`로 관련 아키텍처, 모듈, 계약 문서를 훑는다.
2. 코드와 설정을 변경한다.
3. 기존 문서로 충분하면 `wiki-sync`로 갱신한다.
4. 새 허브나 새 문서 묶음이 필요하면 `wiki-expand`를 추가한다.
5. 마감 전에 `wiki-lint`로 전체 구조를 점검한다.

## 위키 구조 이상

1. 깨진 링크, stale 설명, 충돌 문서를 확인한다.
2. 참조점을 복구해야 하면 `wiki-repair`를 먼저 쓴다.
3. 새 자리가 필요할 때만 `wiki-expand`를 쓴다.
